# GSoC 2026 Participation Checklist — MalariaGEN

Use this checklist to track your progress toward submitting a strong GSoC 2026 proposal with MalariaGEN.

---

## 1. General GSoC Eligibility & Preparation

- [ ] Confirm you meet [GSoC eligibility requirements](https://summerofcode.withgoogle.com/rules)
- [ ] Read the [GSoC Rules](https://summerofcode.withgoogle.com/rules)
- [ ] Read the [GSoC Terms for Participants](https://summerofcode.withgoogle.com/terms/member)
- [ ] Review the [GSoC Timeline](https://developers.google.com/open-source/gsoc/timeline) and note key deadlines
- [ ] Read the [Contributor Guides](https://google.github.io/gsocguides/student/)

---

## 2. Explore MalariaGEN Resources

- [ ] Browse the [MalariaGEN Vector Observatory website](https://www.malariagen.net/vobs)
- [ ] Explore the [malariagen-data-python GitHub repository](https://github.com/malariagen/malariagen-data-python)
- [ ] Review the [training course / analysis documentation](https://anopheles-genomic-surveillance.github.io/home.html)
- [ ] Browse current open issues in the repository to understand technical challenges
- [ ] Read through existing code architecture to understand design patterns

---

## 3. Choose a Project

Pick one of the three available project ideas:

- [ ] **Option A — Large / Hard (350h):** Developing open-source genomic surveillance tools for *Anopheles farauti* in the Pacific
  - Technologies: Python, Statistical Analysis, Google Cloud Storage
  - Mentors: Tristan Dennis, Anastasia Hernandez-Koutoucheva, Jon Brenas

- [ ] **Option B — Medium / Medium (175h):** Building a machine-learning taxon classifier for genomic classification in malaria mosquitoes
  - Technologies: Python, Machine Learning, Google Cloud Storage
  - Mentors: Jon Brenas, Tristan Dennis

- [ ] **Option C — Medium / Medium (175h):** Exploring natural-language interfaces to increase understanding of malaria vector genomic data
  - Technologies: Python, AI, NLP/LLMs, Google Cloud Storage
  - Mentors: Jon Brenas, Chris Clarkson, Anastasia Hernandez-Koutoucheva

---

## 4. Make a Code Contribution (PR)

> **Important:** MalariaGEN will only consider **one PR** during review. Make it count.

- [ ] Fork or clone the [malariagen-data-python repository](https://github.com/malariagen/malariagen-data-python)
- [ ] Set up a local development environment
- [ ] Identify an issue or improvement (existing issue or newly identified)
- [ ] Implement your fix or feature
- [ ] Write/update tests if applicable
- [ ] Open a Pull Request (PR) to the repository
- [ ] Note your PR URL for the application email

---

## 5. Develop Your Proposal Skeleton

- [ ] Define the problem you are solving
- [ ] Outline your proposed approach / solution
- [ ] List deliverables and milestones (aligned with GSoC timeline)
- [ ] Note any dependencies, risks, or unknowns
- [ ] Research related work / prior art

---

## 6. Contact MalariaGEN

Send an email to **support@malariagen.net** with:

- [ ] Subject line includes **GSoC26**
- [ ] Your **GitHub username**
- [ ] **URL to your PR** (the one contribution they will review)
- [ ] Which **project you are interested in** (A, B, or C above)
- [ ] A **skeleton / outline of your proposed approach** for the chosen project

---

## 7. Work with Your Mentor

- [ ] Receive mentor contact from MalariaGEN following your email
- [ ] Discuss and refine your project idea with the mentor
- [ ] Incorporate mentor feedback into your proposal

---

## 8. Submit Your Proposal on GSoC Platform

- [ ] Create an account on [summerofcode.withgoogle.com](https://summerofcode.withgoogle.com/)
- [ ] Submit your proposal before the GSoC application deadline
- [ ] Include all required sections: abstract, deliverables, timeline, biography
- [ ] Double-check that your PR link and GitHub username are included
